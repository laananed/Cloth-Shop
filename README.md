# 服装商城 / 进销存管理系统

## 项目简介

本项目是一个基于 **FastAPI + MySQL + 原生前端** 的服装商城课程设计项目，包含完整的前台购物流程和后台管理流程。

前台用于商品浏览、分类筛选、下单、支付、退款和收货地址管理；后台用于管理员登录、分类管理、商品管理、订单管理、发货处理、退款审核和销量统计。

## 技术栈

- 后端：FastAPI + PyMySQL
- 数据库：MySQL 8.0.28
- 前端：HTML / CSS / JavaScript
- 数据库名：`frieren_cloth_shop_db`

## 主要功能

### 前台功能

- 商品浏览
- 数据库分类导航与搜索组合筛选
- SKU 选择
- 加入购物车
- 购物车勾选下单（支持地址、最多 500 字买家备注和稍后支付）
- 直接购买（支持选填最多 500 字的买家备注）
- 支付
- 取消订单
- 申请退款
- 查看购买记录
- 收货地址管理

### 后台功能

- 管理员登录
- 分类新增、重命名、排序、逻辑删除与恢复
- 调整已有商品分类
- 商品新增
- 商品上下架
- 逻辑删除商品
- 修改 SKU 库存
- 商品筛选和搜索
- 订单查看
- 查看订单详情
- 发货 / 取消发货
- 同意退款 / 拒绝退款
- 销量统计

## 数据库初始化说明

请按项目 SQL 文件编号顺序执行。

当前仓库中可见的 SQL 文件包括：

1. `sql语句/01_数据库结构与增量迁移.sql`
2. `sql语句/02_视图.sql`
3. `sql语句/03_存储过程_触发器_函数.sql`
4. `sql语句/04_测试数据与验证.sql`
5. `sql语句/05_账号与支付密码初始化.sql`
6. `sql语句/06_商品描述增量迁移.sql`
7. `sql语句/07_订单买家备注增量迁移.sql`
8. `sql语句/08_购物车选中项备注下单增量迁移.sql`

`04_测试数据与验证.sql` 会清理并重建固定演示数据，只能在明确的测试数据库执行。已有环境应在确认当前数据库为 `frieren_cloth_shop_db` 后按编号执行尚未应用的增量脚本：`06` 为 `product` 增加可空的普通文本介绍字段并更新商品详情视图；`07` 为 `order_main` 增加可空的 `buyer_remark VARCHAR(500)`，更新订单列表/详情视图，并新增兼容的立即购买存储过程；`08` 保留旧选中项下单过程，并新增支持购物车备注的兼容过程。`06`～`08` 均可重复执行，不会改写已有商品、SKU、库存或订单备注数据。

商品介绍最多 1000 个字符，支持中文与换行，空内容保存为 `NULL`。管理员可通过后台商品卡的“编辑介绍”入口修改或清空，后端接口为 `PATCH /admin/products/{product_id}/description`，需要现有管理员 Bearer 令牌。

立即购买和购物车下单都可选填最多 500 个字符的买家备注，前后端会去除首尾空白，空备注保存为 `NULL`。购物车结算顺序固定为“勾选商品 → 下单 → 选择支付方式 → 输入支付密码”：`POST /orders/from-cart-selected` 先创建 `PENDING_PAYMENT` 订单并只移除选中项，随后 `POST /orders/pay` 支付已有订单；支付失败不会重建订单，用户也可稍后在购买记录中继续支付。备注随各自订单创建事务写入，并在用户端和管理端订单详情中展示。

商品分类直接复用现有 `category` 表和 `product.category_id` 外键，不需要新增 SQL。前台通过 `GET /categories` 读取有效分类；后台通过受管理员令牌保护的 `/admin/categories` 系列接口新增、修改、逻辑删除和恢复分类。有商品引用的分类不能删除，需先在商品管理中调整分类。新增商品必须选择现有有效分类，后端不再根据自由文本自动创建分类。

## 后端启动

进入后端目录后执行：

```bash
cd backend
uvicorn app.main:app --reload --port 8050
```

后端地址：

```text
http://127.0.0.1:8050
```

接口文档地址：

```text
http://127.0.0.1:8050/docs
```

## 常见问题

### 1. 后端端口 8050 被占用怎么办

先关闭已经占用端口的旧后端进程，再重新启动。

也可以先查看端口占用情况，确认是否还有旧窗口没有关闭。

### 2. 前端页面请求后端失败怎么办

先检查后端是否已经启动，并确认地址是否为：

```text
http://127.0.0.1:8050
```

然后打开浏览器开发者工具，查看 `Console` 和 `Network` 中的报错信息。

如果是跨域、接口地址或后端未启动，通常都能在这里看到原因。

### 3. 管理员登录失败怎么办

请先确认账号密码是否正确：

- 账号：`admin@example.com`
- 密码：`admin123456`

如果仍然失败，请检查管理员初始化 SQL 是否已执行完成。

### 4. 图片不显示怎么办

请先确认前端静态服务器已经启动，并检查图片路径是否正确。

如果页面已经刷新但图片仍然不显示，可以清理浏览器缓存后再试。

### 5. 数据库连接失败怎么办

请确认：

- MySQL 已启动
- 数据库 `frieren_cloth_shop_db` 已存在
- 后端配置中的数据库地址、用户名和密码正确
- 本机没有防火墙或权限问题拦截连接
